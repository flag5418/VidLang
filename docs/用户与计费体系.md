# 用户与计费体系（草案）

## 目标

- App 本地支持多用户（学习记录/资源隔离基于本地用户）
- 计费主体只有一个 Supabase 主账号（主管理员），切换主账号即切换计费体系
- 计费规则对用户展示按次费用（人民币），账本简单清晰
- 系统内部可记录真实消耗（token/时长等），用于成本核算与后续调价

## 本地用户模型（SQLite）

### user 表关键字段

- code：本地用户唯一标识（业务主键）
- username：本地登录名；主账号可存 email
- password：本地用户密码（当前为明文/后续可升级 hash）；主账号这里存 passwordHash
- nickname/avatar：展示信息
- email：仅主账号使用
- auth_provider：local / supabase
- supabase_user_id：仅主账号使用（auth.users.id）
- token：仅主账号使用（access_token）

### config 表

- system/current_user_code：最后一次登录的本地用户 code

## 主账号（Supabase）缓存与启动校验

### 缓存策略

- 主账号 email + 明文密码：保存到系统 Keychain/Keystore（flutter_secure_storage）
- 主账号 passwordHash：保存到本地 user.password

### 启动校验逻辑

- 启动读取 system/current_user_code，加载本地 user 到内存（AppConfig.currentUser）
- 若当前用户为主账号（auth_provider=supabase）
  - 读取安全存储中的 email+password
  - 静默调用 Supabase 登录校验
  - 成功：刷新本地主账号的 token / supabase_user_id / email 等信息
  - 失败：进入登录页要求用户手动登录主账号
- 若当前用户为本地账号（auth_provider=local）
  - 仍会用安全存储里的主账号 email+password 静默校验一次（用于保证计费主体可用）
  - 校验失败不影响进入首页，但后续付费能力会因主账号不可用而失败并提示重新登录

## 主账号唯一性与切换规则

- 本地只允许存在一个 auth_provider=supabase 的用户记录（主管理员）
- 当用户用新的邮箱登录/注册 Supabase 主账号时
  - UI 弹窗提醒“切换主账号将切换计费体系”
  - 确认后覆盖本地主账号信息（不删除本地子用户）
- 历史账单永久归属产生当时的 supabase_user_id，不做迁移

## Supabase（计费侧）核心表

### pricing_rule（按次计费规则）

- id
- owner_supabase_user_id（主账号 auth.users.id）
- scene（模块，可选）
- action（功能点：translate/tts/ai/…）
- entry（入口，可选）
- model
- price_cny_per_request（按次金额）
- effective_from
- status

### usage_event（逐笔消费明细）

- id
- owner_supabase_user_id
- local_user_code
- local_user_name（昵称快照）
- rule_id
- scene/action/entry
- model
- request_id（幂等键，唯一）
- cost_cny（按次扣费结果）
- created_at
- meta（JSON：可包含 token/耗时/供应商返回信息等内部证据）

### usage_daily_tokens_agg（按天汇总）

- owner_supabase_user_id
- local_user_code
- date
- model
- feature（同 action）
- count
- prompt_tokens
- completion_tokens
- total_tokens
- total_cost_cny
- 唯一键：(owner_supabase_user_id, local_user_code, date, model, feature)

### topup_record（充值记录）

- id
- owner_supabase_user_id
- amount_cny
- note
- channel
- status
- created_at / paid_at

### wallet_ledger（余额流水）

- id
- owner_supabase_user_id
- type（topup/consume/adjust/refund）
- amount_cny（消费为负）
- ref_id（关联 usage_event 或 topup_record）
- created_at

## Edge Function（本地管理与调试）

- 目录：supabase/functions/\*
- 每个能力建议对应一个函数入口（例如 translate / tts / ai_chat / usage_event）
- 计费建议在 Edge Function 侧完成：
  - 调用模型 → 解析真实消耗 → 写 usage_event（幂等）→ 写 wallet_ledger 扣费 → upsert 日汇总
