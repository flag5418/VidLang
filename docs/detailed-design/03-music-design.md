# VidLang 详细设计 — 歌曲/MTV 学习功能

## 一、核心结论

### MTV 学习方案 = 戴耳机跟唱评分

> 用户戴耳机听 MTV → 耳机麦克风采到的主要是用户自己的声音（耳机漏音极低）→ 录音上传声通英文评分

**不需要做卡拉OK式的人声分离。**

### 技术可行性确认

```
用户播放 MTV（正常音量 100%）
    ↓
用户戴耳机听原唱
    ↓
用户点击录音 → 系统开始录音
    ↓
录音中：耳机麦克风只采到用户跟唱声音（原唱通过耳机播放，不外放）
    ↓
录音停止 → 音频文件 = 干净的跟唱人声
    ↓
上传声通英文评分 → 发音/流利度/完整度评分
    ↓
展示逐词评分反馈
```

**关键区别：录音时不需要调低视频音量**（和普通视频跟读不同，普通跟读需要降原声避免录音混入）。

## 二、数据模型

MTV 在 VidLang 中 = **带歌词时间轴的视频文件**。

所以不需要独立模型，直接扩展现有 VideoInfo：

```dart
// VideoInfo 扩展字段
class VideoInfo extends BaseEntity {
  // ... 原有字段 ...

  // 新增 MTV 相关字段
  bool hasLyrics;           // 是否有歌词时间轴
  bool isMusicVideo;        // 是否是 MTV
  String? artist;           // 歌手名
  String? genre;            // 风格
}
```

歌词存储在现有 Subtitles 表，用 type 字段区分：

```dart
class Subtitles extends BaseEntity {
  String videoCode;
  String type;              // 'subtitle' 或 'lyric'
  int startPositionMs;
  int endPositionMs;
  String content;
  String? contentTranslate;
  int sequenceIndex;
}
```

同一视频可以同时有字幕和歌词——用户在播放器中切换。

## 三、歌词导入方式

### 3.1 SRT/LRC 格式导入

导入 MTV 视频时，同时导入配套的 SRT 字幕文件（歌词版）或 LRC 歌词文件。

SRT 格式：
```
1
00:00:02,000 --> 00:00:06,000
Let it be, let it be, let it be, let it be

2
00:00:06,500 --> 00:00:10,000
Whisper words of wisdom, let it be
```

LRC 格式：
```
[00:02.00]Let it be, let it be, let it be, let it be
[00:06.50]Whisper words of wisdom, let it be
```

### 3.2 粘贴歌词 + 自动对齐

用户粘贴歌词文本，系统根据视频时长自动估算每句歌词的时间戳。

### 3.3 直接使用视频英文字幕

用户选择"将此视频作为 MTV 学习"→ 已有英文字幕自动作为歌词。

## 四、播放器交互

### 4.1 歌词模式

PlayerPage 增加"歌词模式/字幕模式"切换：

```
┌─────────────────────────────────────────────────────┐
│  [←] Let It Be — The Beatles               [🎤]   │
├─────────────────────────────────────────────────────┤
│                                                      │
│          视频画面（歌词模式时背景化）                  │
│                                                      │
│  ┌─────────────────────────────────────────────┐   │
│  │       Let it be,                            │   │ ← 当前歌词高亮
│  │    let it be, let it be                     │   │
│  │                                             │   │
│  │  Whisper words of wisdom                    │   │ ← 下一句（淡出）
│  └─────────────────────────────────────────────┘   │
│                                                      │
├─────────────────────────────────────────────────────┤
│  ◀  ▶/⏸  ▶  x1.0  [Tr] [Fol] [Sub/Lyric] [Quiz] │
└─────────────────────────────────────────────────────┘
```

### 4.2 和视频字幕模式的关系

| 模式 | 显示内容 | 适用场景 |
|------|---------|---------|
| 字幕（Subtitle） | 对话/旁白字幕 | 看美剧/纪录片 |
| 歌词（Lyrics） | 歌词时间轴 | 听英文歌/MTV |

切换按钮：底部 `[Sub/Lyrics]` 切换

### 4.3 跟唱面板

跟唱和视频跟读完全一致，唯一的区别是录音时**不降视频音量**：

```
┌──────────────────────────────────────────────┐
│  Follow Along — Line 3/24                   │
├──────────────────────────────────────────────┤
│                                               │
│  "Let it be, let it be..."                   │
│                                               │
│  🔊 [Listen]  🎤 [Record]  ▶ [Playback]     │
│                                               │
│  Score: 88/100                                │
│  Accuracy: 92%  Fluency: 85%                 │
│                                               │
│  ◀ Line 3/24 ▶                               │
└──────────────────────────────────────────────┘
```

评分引擎：**声通英文发音评分**（不是音乐评分）

## 五、和视频跟读的差异对比

| 维度 | 视频跟读 | MTV 跟唱 |
|------|---------|---------|
| 内容 | 对话/独白字幕 | 歌词时间轴 |
| 耳朵听 | 视频原声（对话） | 原唱歌曲 |
| 嘴巴跟 | 跟读重复 | 跟唱 |
| 录音时音量 | 调低视频音量（降原声） | **不调低**（戴耳机听） |
| 评分引擎 | 声通英文发音评分 | 声通英文发音评分 |
| 评分维度 | 发音/流利度/完整度 | 发音/流利度/完整度 |
| UI 面板 | DubPanel | FollowPanel（复用） |
| 查词 | 点击字幕单词 | 点击歌词单词 |
| 测试 | 字幕内容出题 | 歌词内容出题 |

**核心结论：MTV 在代码层面 ≈ 视频的一个子类型，标签为 isMusicVideo=true 的 VideoInfo。**

---

## 六、MTV 功能总结

| 维度 | 方案 |
|------|------|
| **播放** | 和视频共用 PlayerPage |
| **内容** | VideoInfo + Subtitles(type=lyric) |
| **歌词** | 和字幕共用 Subtitles 表，type 区分 |
| **显示** | PlayerPage 切换"字幕"/"歌词"模式 |
| **跟唱** | 复用 FollowPanel，录音不调低音量 |
| **评分** | 声通英文发音评分（无需音乐评分） |
| **查词** | 点击歌词单词 → WordCard |
| **测试** | 复用统一测试引擎，数据源 = 歌词句子 |
