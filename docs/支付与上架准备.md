# 支付与上架准备

## 一、概述

vidlang 的支付策略分平台处理：

| 平台 | 安装来源 | 支付方式 |
|------|---------|---------|
| iOS | App Store | StoreKit IAP（固定金额档位） |
| Android | Google Play | Google Play Billing（固定金额档位） |
| Android | APK 侧载（国内用户） | Edge Function + 支付宝 H5 支付（备选） |

## 二、开发测试环境配置

### 2.1 目标

让 Android 模拟器能访问 Google 服务，用于测试 Google Play Billing。

### 2.2 FlClash 配置

1. 打开 FlClash → 设置
2. 找到 **"允许局域网连接"（Allow LAN）**，打开
3. 确认代理端口（默认通常如下）：

| 协议 | 默认端口 |
|------|---------|
| HTTP 混合代理 | `7897` |
| HTTP 单独 | `7890` |
| SOCKS5 | `7891` |

### 2.3 创建带 Google Play 的 Android 模拟器

```
Android Studio → Device Manager → Create Device
→ 选择 Pixel 系列（如 Pixel 6）
→ 选择系统镜像：带 "(Google Play)" 字样的（如 API 34 Google Play）
→ 完成创建
```

首次启动后 Google Play 会自动安装，无需手动刷入 GMS。

### 2.4 配置模拟器网络代理

**方法 A：命令行启动（推荐）**

```bash
# 查看已有模拟器
emulator -list-avds

# 启动并指定代理（7897 替换为你的 FlClash 端口）
emulator -avd Pixel_6_API_34 -http-proxy http://10.0.2.2:7897
```

**方法 B：系统设置**

```
模拟器内 Settings → Network & Internet → WiFi →
长按 "AndroidWifi" → Modify Network →
Proxy: Manual
Host: 10.0.2.2   （指向宿主机）
Port: 7897       （FlClash 端口）
```

**网络拓扑说明：**

```
宿主机 FlClash（开启 Allow LAN）
       │
       │ 10.0.2.2:7897
       ▼
Android 模拟器（10.0.2.2 是宿主机地址）
       │
       ▼
Google Play / Google Play Billing（正常工作）
```

### 2.5 验证

模拟器内：
- 打开浏览器访问 `google.com` → 能打开则网络通
- 打开 Play Store → 登录 Google 账号 → 能正常访问则 OK

### 2.6 备用方案：海外真机

如果模拟器不方便，可以考虑买一台海外版 Android 真机：

| 机型 | 二手价格 | 说明 |
|------|---------|------|
| Google Pixel 4a | 500-800 元 | 原生 GMS |
| Google Pixel 5 | 800-1200 元 | 原生 GMS |
| Google Pixel 6 | 1200-1800 元 | 原生 GMS |
| 海外版小米/三星 | 视型号 | 需确认有 GMS |

不插 SIM 卡也能正常使用 Play Store 和测试 IAP。

## 三、银行卡与支付准备

### 3.1 各场景对银行卡的要求

| 场景 | 费用 | 支持的支付方式 | 建行纯银联卡 | 建行双标卡 | 招行 Visa 全币卡 |
|------|------|--------------|------------|-----------|----------------|
| Apple Developer 年费 | $99/年 | Visa/MC/Amex/支付宝 | ❌ | ⚠️ 可能可以 | ✅ |
| Google Play Console 注册 | $25 一次性 | Visa/MC/Amex | ❌ | ⚠️ 常被风控 | ✅ |
| App Store IAP 收款 | — | 提现时用 | 需单独设置 | 需单独设置 | 需单独设置 |
| Google Play 收款 | — | 提现时用 | 需单独设置 | 需单独设置 | 需单独设置 |

### 3.2 推荐方案

**最稳妥的配置：招商银行 Visa 全币卡**

- 免费申请（招商银行 App 或网点）
- 一周左右到卡
- 专门用于海外支付，不易被风控拦截
- 无年费

**Apple Developer 年费备选：**
- Apple 支持**支付宝**支付 $99 年费
- 如果 Visa 卡不方便，直接用支付宝

### 3.3 收款账户（后续提现用）

- iOS App Store 销售收益 → 绑定你的银行账户（公司对公或个人均可）
- Google Play 销售收益 → 绑定银行账户
- 个人开发者每年有 **5 万美金结汇额度**，超过需提供贸易背景材料
- 公司账户无此限制，需提供 SWIFT Code

## 四、个人 vs 公司：账号类型选择

### 4.1 对比

| 对比维度 | **个人开发者** | **小公司** |
|---------|-------------|-----------|
| Apple 年费 | $99 相同 | $99 相同 |
| Google 注册费 | $25 相同 | $25 相同 |
| 商店显示名称 | 个人姓名 | 公司名称 |
| 注册审核时间 | 1-2 天 | 1-2 周（需 DUNS 码） |
| DUNS 码 | ❌ 不需要 | ✅ 需要（免费申请） |
| 上架审核标准 | 相同 | 相同 |
| 税务表格 | W-8BEN（简单） | W-8BEN-E（略复杂） |
| 收款账户 | 个人银行卡 | 对公账户 |
| 法律风险 | 个人承担 | 公司有限责任 |
| 开票 | 个人无法开专票 | 可开票 |
| 维护成本 | 低 | 公司年检、记账 |

### 4.2 建议

**推荐用公司资质申请。** 理由：

1. App Store 商店页显示公司名比个人名更有说服力，尤其教育/学习类产品
2. 公司对公账户收美元比个人结汇方便（个人有 5 万美金/年额度限制）
3. 法律保护：应用出了版权/内容问题，公司承担有限责任
4. 你有实体公司，申请 DUNS 码不额外花钱

**唯一的额外步骤：申请 DUNS 码**（邓白氏编码），免费，耗时 1-2 周。

## 七、支付流程架构

```
Flutter App
    │
    ├─ 调用 in_app_purchase 发起购买
    │  （Google Play Billing / StoreKit）
    │
    ├─ 支付成功，拿到 purchaseToken / receipt
    │
    ├─ supabase.functions.invoke('verify_purchase', {
    │     platform: 'google_play' | 'app_store',
    │     productId: 'topup_30',
    │     token: purchaseToken,
    │   })
    │
    ├─ Edge Function 向 Google/Apple 服务器验证收据
    │
    ├─ 验证通过 → INSERT topup_record + wallet_ledger
    │
    └─ 返回成功 → Flutter 更新余额 UI

备选：APK 侧载用户
    └─ Flutter 打开 H5 支付宝页面
       → Edge Function 创建订单
       → 用户扫码付款
       → 支付宝异步回调
       → Edge Function 入账
```

## 八、常见问题

### Q：国内手机没有 GMS，能否测试 Google Play Billing？

不能。Google Play Billing 依赖 Google Play Services。必须用海外设备或带 Google Play 的模拟器。

### Q：Google Play Billing 的测试是否需要上传 APK 到 Play Console？

是的。需要在 Play Console 创建应用 → 上传到内部测试轨道（Internal Testing）→ 添加测试者邮箱 → 测试者通过 Play Store 安装才能触发真实的 Billing 流程。模拟器上的 License Testing 可配置为不扣费。

### Q：侧载 APK 的用户如何付费？

侧载 APK 无法使用 Google Play Billing。备选方案是支付宝 H5 支付，通过 Edge Function 中转入账。

### Q：公司注销了，App 会受影响吗？

会。Apple 和 Google 会定期验证公司资质。公司注销后账号可能被冻结。建议长期稳定运营再决定用公司还是个人。
